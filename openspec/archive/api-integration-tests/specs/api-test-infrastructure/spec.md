## ADDED Requirements

### API_TEST_INF-001: 测试基础设施必须提供 HTTP 客户端封装
测试框架必须提供统一的 HTTP 请求封装，支持 GET、POST、PUT、DELETE、PATCH 方法。

#### 场景:发送 GET 请求
- **当** 测试代码调用 HTTP GET 方法
- **那么** 系统必须返回 HTTP 响应和错误信息

#### 场景:发送 POST 请求
- **当** 测试代码调用 HTTP POST 方法并携带 JSON 请求体
- **那么** 系统必须正确发送请求并返回响应

#### 场景:发送带 Header 的请求
- **当** 测试代码调用 HTTP 方法并携带自定义 Header
- **那么** 系统必须在请求中包含指定 Header

### API_TEST_INF-002: 测试基础设施必须支持 JWT Token 管理
测试框架必须支持从登录响应中提取 Token，并在后续请求中自动附加 Authorization Header。

#### 场景:提取 Access Token
- **当** 用户登录成功
- **那么** 系统必须能够从响应中提取 access_token

#### 场景:自动附加 Token
- **当** 测试代码设置已登录用户 Token
- **那么** 后续 HTTP 请求必须自动包含 Authorization: Bearer <token> Header

### API_TEST_INF-003: 测试基础设施必须提供响应验证工具
测试框架必须提供验证 HTTP 响应码、响应体结构、JSON 字段的辅助函数。

#### 场景:验证状态码
- **当** 收到 HTTP 响应
- **那么** 系统必须能够验证状态码是否符合预期

#### 场景:验证 JSON 字段
- **当** 收到 JSON 响应
- **那么** 系统必须能够验证指定字段的值

### API_TEST_INF-004: 测试基础设施必须提供测试数据生成工具
测试框架必须提供生成随机用户名、邮箱、文章数据等的工具函数，避免数据冲突。

#### 场景:生成随机用户名
- **当** 调用用户名生成函数
- **那么** 系统必须返回唯一的随机用户名

#### 场景:生成测试文章数据
- **当** 调用文章数据生成函数
- **那么** 系统必须返回包含标题、内容、标签的完整测试数据

### API_TEST_INF-005: 测试基础设施必须提供数据清理机制
每个测试用例执行后必须清理创建的测试数据，保证测试隔离性。

#### 场景:清理测试用户
- **当** 测试用例执行完成
- **那么** 系统必须删除该测试创建的用户数据

#### 场景:清理测试文章
- **当** 测试用例执行完成
- **那么** 系统必须删除该测试创建的文章及相关数据
