## ADDED Requirements

### COMMENT_TEST-001: 创建评论接口必须正确工作
POST /api/v1/comments 接口必须允许登录用户对文章发表评论。

#### 场景:成功创建评论
- **当** 已登录用户发送有效的评论数据（文章ID、内容）
- **那么** 系统必须创建评论并返回评论详情
- **并且** 文章评论数必须增加

#### 场景:未登录用户创建评论
- **当** 未携带 Token 发送创建请求
- **那么** 系统必须返回 HTTP 401 状态码

#### 场景:评论不存在的文章
- **当** 发送文章 ID 不存在的评论请求
- **那么** 系统必须返回 HTTP 400 状态码

### COMMENT_TEST-002: 获取评论列表接口必须正确工作
GET /api/v1/comments 接口必须返回指定文章的评论列表。

#### 场景:获取文章评论列表
- **当** 发送 article_id 参数的请求
- **那么** 系统必须返回该文章的评论列表
- **并且** 支持分页参数

#### 场景:缺少文章 ID 参数
- **当** 发送缺少 article_id 的请求
- **那么** 系统必须返回 HTTP 400 状态码

### COMMENT_TEST-003: 更新评论接口必须正确工作
PUT /api/v1/comments/:id 接口必须允许用户更新自己的评论。

#### 场景:成功更新评论
- **当** 评论作者发送更新请求
- **那么** 系统必须更新评论内容并返回成功

#### 场景:非作者更新评论
- **当** 非评论作者的用户发送更新请求
- **那么** 系统必须返回 HTTP 403 状态码

#### 场景:更新不存在的评论
- **当** 发送更新不存在评论 ID 的请求
- **那么** 系统必须返回 HTTP 400 状态码

### COMMENT_TEST-004: 删除评论接口必须正确工作
DELETE /api/v1/comments/:id 接口必须允许用户删除自己的评论。

#### 场景:成功删除评论
- **当** 评论作者发送删除请求
- **那么** 系统必须删除评论并返回成功
- **并且** 文章评论数必须减少

#### 场景:非作者删除评论
- **当** 非评论作者的用户发送删除请求
- **那么** 系统必须返回 HTTP 403 状态码

#### 场景:删除不存在的评论
- **当** 发送删除不存在评论 ID 的请求
- **那么** 系统必须返回 HTTP 400 状态码
