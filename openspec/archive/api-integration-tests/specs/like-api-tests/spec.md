## ADDED Requirements

### LIKE_TEST-001: 点赞接口必须正确工作
POST /api/v1/likes 接口必须允许登录用户对文章或评论点赞。

#### 场景:成功点赞文章
- **当** 已登录用户发送 target_type=article&target_id=xxx 的点赞请求
- **那么** 系统必须记录点赞并返回成功
- **并且** 文章点赞数必须增加

#### 场景:成功点赞评论
- **当** 已登录用户发送 target_type=comment&target_id=xxx 的点赞请求
- **那么** 系统必须记录点赞并返回成功

#### 场景:重复点赞
- **当** 用户对已点赞的内容再次点赞
- **那么** 系统必须返回适当错误码或忽略重复操作

#### 场景:未登录用户点赞
- **当** 未携带 Token 发送点赞请求
- **那么** 系统必须返回 HTTP 401 状态码

#### 场景:点赞不存在的内容
- **当** 发送不存在的文章或文章 ID 的点赞请求
- **那么** 系统必须返回 HTTP 400 状态码

### LIKE_TEST-002: 取消点赞接口必须正确工作
DELETE /api/v1/likes 接口必须允许登录用户取消点赞。

#### 场景:成功取消点赞
- **当** 已登录用户发送取消点赞请求
- **那么** 系统必须删除点赞记录并返回成功
- **并且** 内容点赞数必须减少

#### 场景:取消未点赞的内容
- **当** 用户取消一个从未点赞过的内容
- **那么** 系统必须返回成功或适当提示

#### 场景:未登录用户取消点赞
- **当** 未携带 Token 发送取消点赞请求
- **那么** 系统必须返回 HTTP 401 状态码

### LIKE_TEST-003: 获取点赞状态接口必须正确工作
GET /api/v1/likes/status 接口必须返回用户对指定内容的点赞状态。

#### 场景:获取已点赞状态
- **当** 已登录用户查询已点赞的内容
- **那么** 系统必须返回 is_liked=true

#### 场景:获取未点赞状态
- **当** 已登录用户查询未点赞的内容
- **那么** 系统必须返回 is_liked=false

#### 场景:获取点赞数
- **当** 查询点赞状态
- **那么** 系统必须返回该内容的总点赞数

### LIKE_TEST-004: 获取用户点赞列表接口必须正确工作
GET /api/v1/user/likes 接口必须返回当前用户的点赞历史。

#### 场景:获取点赞的文章列表
- **当** 已登录用户发送 target_type=article 的请求
- **那么** 系统必须返回该用户点赞的文章列表

#### 场景:获取点赞的评论列表
- **当** 已登录用户发送 target_type=comment 的请求
- **那么** 系统必须返回该用户点赞的评论列表

#### 场景:分页获取点赞列表
- **当** 发送 page 和 size 参数
- **那么** 系统必须按分页返回结果
