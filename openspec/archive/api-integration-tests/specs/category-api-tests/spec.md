## ADDED Requirements

### CATEGORY_TEST-001: 获取栏目列表接口必须正确工作
GET /api/v1/categories 接口必须返回所有栏目列表。

#### 场景:获取所有栏目
- **当** 发送获取栏目列表请求
- **那么** 系统必须返回所有栏目的基本信息
- **并且** 包含栏目 ID、名称、描述等

### CATEGORY_TEST-002: 获取栏目详情接口必须正确工作
GET /api/v1/categories/:id 接口必须返回指定栏目的详细信息。

#### 场景:获取存在的栏目详情
- **当** 发送已存在栏目 ID 的请求
- **那么** 系统必须返回该栏目的完整信息

#### 场景:获取不存在的栏目
- **当** 发送不存在栏目 ID 的请求
- **那么** 系统必须返回 HTTP 404 状态码

### CATEGORY_TEST-003: 获取栏目下文章接口必须正确工作
GET /api/v1/categories/:id/articles 接口必须返回指定栏目下的文章列表。

#### 场景:获取栏目文章列表
- **当** 发送栏目 ID 的请求
- **那么** 系统必须返回该栏目下的公开文章列表
- **并且** 支持分页参数

#### 场景:获取不存在栏目的文章
- **当** 发送不存在栏目 ID 的请求
- **那么** 系统必须返回 HTTP 404 状态码

### CATEGORY_TEST-004: 创建栏目接口必须正确工作
POST /api/v1/categories 接口必须允许登录用户创建新栏目。

#### 场景:成功创建栏目
- **当** 已登录用户发送有效的栏目数据（名称）
- **那么** 系统必须创建栏目并返回成功

#### 场景:未登录用户创建栏目
- **当** 未携带 Token 发送创建请求
- **那么** 系统必须返回 HTTP 401 状态码

#### 场景:创建重复名称栏目
- **当** 发送已存在名称的栏目
- **那么** 系统必须返回 HTTP 400 状态码

### CATEGORY_TEST-005: 更新栏目接口必须正确工作
PUT /api/v1/categories/:id 接口必须允许用户更新栏目信息。

#### 场景:成功更新栏目
- **当** 已登录用户发送更新请求
- **那么** 系统必须更新栏目信息并返回成功

#### 场景:更新不存在的栏目
- **当** 发送更新不存在栏目 ID 的请求
- **那么** 系统必须返回 HTTP 404 状态码

### CATEGORY_TEST-006: 删除栏目接口必须正确工作
DELETE /api/v1/categories/:id 接口必须允许用户删除栏目。

#### 场景:成功删除栏目
- **当** 已登录用户发送删除请求
- **那么** 系统必须删除栏目并返回成功

#### 场景:删除不存在的栏目
- **当** 发送删除不存在栏目 ID 的请求
- **那么** 系统必须返回 HTTP 404 状态码

### CATEGORY_TEST-007: 添加文章到栏目接口必须正确工作
POST /api/v1/articles/:id/categories 接口必须允许将文章添加到栏目。

#### 场景:成功添加文章到栏目
- **当** 文章作者发送添加请求
- **那么** 系统必须建立文章与栏目的关联

#### 场景:非作者添加文章到栏目
- **当** 非文章作者发送添加请求
- **那么** 系统必须返回 HTTP 403 状态码

### CATEGORY_TEST-008: 获取文章所属栏目接口必须正确工作
GET /api/v1/articles/:id/categories 接口必须返回文章所属的栏目列表。

#### 场景:获取文章的栏目列表
- **当** 发送文章 ID 的请求
- **那么** 系统必须返回该文章所属的所有栏目
