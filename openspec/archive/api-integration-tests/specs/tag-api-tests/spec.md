## ADDED Requirements

### TAG_TEST-001: 获取所有标签接口必须正确工作
GET /api/v1/tags 接口必须返回系统中所有标签列表。

#### 场景:获取所有标签
- **当** 发送获取标签列表请求
- **那么** 系统必须返回所有标签的列表
- **并且** 包含标签 ID、名称、文章数量等

### TAG_TEST-002: 获取标签下文章接口必须正确工作
GET /api/v1/tags/:id/articles 接口必须返回指定标签下的文章列表。

#### 场景:获取标签文章列表
- **当** 发送标签 ID 的请求
- **那么** 系统必须返回包含该标签的公开文章列表
- **并且** 支持分页参数

#### 场景:获取不存在标签的文章
- **当** 发送不存在标签 ID 的请求
- **那么** 系统必须返回适当的错误码

### TAG_TEST-003: 创建标签接口必须正确工作
POST /api/v1/tags 接口必须允许登录用户创建新标签。

#### 场景:成功创建标签
- **当** 已登录用户发送有效的标签数据（名称）
- **那么** 系统必须创建标签并返回标签详情

#### 场景:未登录用户创建标签
- **当** 未携带 Token 发送创建请求
- **那么** 系统必须返回 HTTP 401 状态码

#### 场景:创建重复名称标签
- **当** 发送已存在名称的标签
- **那么** 系统必须返回 HTTP 400 状态码或返回已存在的标签

### TAG_TEST-004: 更新标签接口必须正确工作
PUT /api/v1/tags/:id 接口必须允许用户更新标签信息。

#### 场景:成功更新标签
- **当** 已登录用户发送更新请求
- **那么** 系统必须更新标签信息并返回成功

#### 场景:更新不存在的标签
- **当** 发送更新不存在标签 ID 的请求
- **那么** 系统必须返回 HTTP 400 状态码

### TAG_TEST-005: 删除标签接口必须正确工作
DELETE /api/v1/tags/:id 接口必须允许用户删除标签。

#### 场景:成功删除标签
- **当** 已登录用户发送删除请求
- **那么** 系统必须删除标签并返回成功

#### 场景:删除不存在的标签
- **当** 发送删除不存在标签 ID 的请求
- **那么** 系统必须返回 HTTP 400 状态码

### TAG_TEST-006: 获取作者标签接口必须正确工作
GET /api/v1/author/tags 接口必须返回当前作者使用的所有标签。

#### 场景:获取作者标签列表
- **当** 已登录作者发送请求
- **那么** 系统必须返回该作者文章使用的所有标签

#### 场景:未登录用户获取作者标签
- **当** 未携带 Token 发送请求
- **那么** 系统必须返回 HTTP 401 状态码
