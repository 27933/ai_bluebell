## ADDED Requirements

### ARTICLE_TEST-001: 获取文章列表接口必须正确工作
GET /api/v1/articles 接口必须返回公开文章列表，支持分页和排序。

#### 场景:获取默认文章列表
- **当** 发送无参数的请求
- **那么** 系统必须返回第 1 页的文章列表，每页 20 条
- **并且** 只返回已发布状态的文章

#### 场景:获取指定页码和数量
- **当** 发送 page=2&size=10 的请求
- **那么** 系统必须返回第 2 页的 10 条文章

#### 场景:按时间排序
- **当** 发送 sort=time 的请求
- **那么** 系统必须按发布时间降序返回文章

#### 场景:按热度排序
- **当** 发送 sort=hot 的请求
- **那么** 系统必须按热度（浏览量、点赞数）排序返回文章

### ARTICLE_TEST-002: 获取文章详情接口必须正确工作
GET /api/v1/articles/:id 接口必须返回指定文章的详细信息。

#### 场景:获取存在的文章详情
- **当** 发送已存在文章 ID 的请求
- **那么** 系统必须返回该文章的完整信息
- **并且** 包含作者信息、标签、统计数等

#### 场景:获取不存在的文章
- **当** 发送不存在文章 ID 的请求
- **那么** 系统必须返回 HTTP 404 或适当错误码

#### 场景:获取文章时记录访问
- **当** 获取文章详情
- **那么** 系统必须异步记录访问日志

### ARTICLE_TEST-003: 创建文章接口必须正确工作
POST /api/v1/articles 接口必须允许登录用户创建新文章。

#### 场景:成功创建文章
- **当** 已登录用户发送有效的文章数据（标题、内容）
- **那么** 系统必须创建文章并返回文章详情
- **并且** 文章作者必须是当前用户

#### 场景:未登录用户创建文章
- **当** 未携带 Token 发送创建请求
- **那么** 系统必须返回 HTTP 401 状态码

#### 场景:创建文章时缺少必填字段
- **当** 发送缺少标题或内容的请求
- **那么** 系统必须返回 HTTP 400 状态码

### ARTICLE_TEST-004: 更新文章接口必须正确工作
PUT /api/v1/author/articles/:id 接口必须允许作者更新自己的文章。

#### 场景:成功更新文章
- **当** 文章作者发送更新请求
- **那么** 系统必须更新文章内容并返回成功

#### 场景:非作者更新文章
- **当** 非文章作者的用户发送更新请求
- **那么** 系统必须返回 HTTP 403 状态码

### ARTICLE_TEST-005: 删除文章接口必须正确工作
DELETE /api/v1/author/articles/:id 接口必须允许作者删除自己的文章。

#### 场景:成功删除文章
- **当** 文章作者发送删除请求
- **那么** 系统必须删除文章并返回成功

#### 场景:删除不存在的文章
- **当** 发送删除不存在文章 ID 的请求
- **那么** 系统必须返回适当的错误码

### ARTICLE_TEST-006: 文章状态管理接口必须正确工作
PATCH /api/v1/author/articles/:id/status 接口必须允许作者更新文章状态。

#### 场景:发布草稿文章
- **当** 作者将文章状态从 draft 更新为 published
- **那么** 系统必须更新状态并返回成功

#### 场景:下线已发布文章
- **当** 作者将文章状态从 published 更新为 offline
- **那么** 系统必须更新状态并返回成功

### ARTICLE_TEST-007: 获取精选文章接口必须正确工作
GET /api/v1/articles/featured 接口必须返回精选文章列表。

#### 场景:获取默认数量精选文章
- **当** 发送无参数的请求
- **那么** 系统必须返回精选文章列表（默认3条）

#### 场景:获取指定数量精选文章
- **当** 发送 limit=5 的请求
- **那么** 系统必须返回最多 5 条精选文章

### ARTICLE_TEST-008: 搜索文章接口必须正确工作
GET /api/v1/articles/search 接口必须支持按关键词、作者、标签搜索文章。

#### 场景:按关键词搜索
- **当** 发送 keyword=golang 的搜索请求
- **那么** 系统必须返回标题或内容包含该关键词的文章

#### 场景:按作者搜索
- **当** 发送 author_name=testuser 的搜索请求
- **那么** 系统必须返回该作者的文章

#### 场景:按标签搜索
- **当** 发送 tag=backend 的搜索请求
- **那么** 系统必须返回包含该标签的文章

### ARTICLE_TEST-009: 文章统计接口必须正确工作
GET /api/v1/article-stats/daily 接口必须返回文章的日访问统计。

#### 场景:获取默认天数统计
- **当** 发送文章 ID 的请求
- **那么** 系统必须返回该文章最近 30 天的访问数据

#### 场景:获取指定天数统计
- **当** 发送 days=7 的请求
- **那么** 系统必须返回该文章最近 7 天的访问数据
