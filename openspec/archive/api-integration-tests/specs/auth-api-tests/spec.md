## ADDED Requirements

### AUTH_TEST-001: 用户注册接口必须正确工作
POST /api/v1/auth/signup 接口必须正常处理用户注册请求。

#### 场景:成功注册新用户
- **当** 发送包含有效用户名、密码、确认密码的注册请求
- **那么** 系统必须返回 HTTP 200 状态码和成功消息
- **并且** 用户数据必须被保存到数据库

#### 场景:注册时用户名已存在
- **当** 发送已存在用户名的注册请求
- **那么** 系统必须返回 HTTP 400 状态码和错误信息

#### 场景:注册时密码不匹配
- **当** 发送密码和确认密码不一致的注册请求
- **那么** 系统必须返回 HTTP 400 状态码和验证错误

### AUTH_TEST-002: 用户登录接口必须正确工作
POST /api/v1/auth/login 接口必须正常处理用户登录请求并返回 Token。

#### 场景:成功登录
- **当** 发送正确的用户名和密码
- **那么** 系统必须返回 HTTP 200 状态码
- **并且** 响应必须包含用户信息和 access_token、refresh_token

#### 场景:登录时密码错误
- **当** 发送正确用户名但错误密码
- **那么** 系统必须返回 HTTP 401 状态码

#### 场景:登录时用户不存在
- **当** 发送不存在的用户名
- **那么** 系统必须返回 HTTP 401 状态码

### AUTH_TEST-003: Token 刷新接口必须正确工作
POST /api/v1/auth/refresh 接口必须使用 refresh_token 换取新的 access_token。

#### 场景:成功刷新 Token
- **当** 发送有效的 refresh_token
- **那么** 系统必须返回新的 access_token

#### 场景:使用无效 Token 刷新
- **当** 发送无效的 refresh_token
- **那么** 系统必须返回 HTTP 401 状态码

### AUTH_TEST-004: 获取用户信息接口必须正确工作
GET /api/v1/auth/profile 接口必须返回当前登录用户的详细信息。

#### 场景:已登录用户获取信息
- **当** 已登录用户发送请求获取个人信息
- **那么** 系统必须返回该用户的完整信息

#### 场景:未登录用户获取信息
- **当** 未携带 Token 发送请求
- **那么** 系统必须返回 HTTP 401 状态码

### AUTH_TEST-005: 更新用户信息接口必须正确工作
PUT /api/v1/auth/profile 接口必须允许用户更新个人信息。

#### 场景:成功更新用户信息
- **当** 已登录用户发送更新请求（如修改昵称）
- **那么** 系统必须更新用户信息并返回成功
- **并且** 再次获取用户信息时应看到更新后的值

#### 场景:未登录用户更新信息
- **当** 未携带 Token 发送更新请求
- **那么** 系统必须返回 HTTP 401 状态码
