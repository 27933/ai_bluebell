## 为什么

Bluebell 博客系统目前缺少全面的 API 集成测试，导致接口变更时难以快速验证功能完整性。随着系统功能扩展（文章、评论、点赞、栏目、管理员等模块），需要一套完整的自动化测试来保障代码质量和接口稳定性。

## 变更内容

1. **创建测试目录结构**：在 `tests/api/` 目录下组织所有 API 集成测试
2. **编写全面测试用例**：覆盖所有 40+ 个 API 接口的正向和异常场景
3. **测试基础设施**：提供测试工具函数、数据准备、环境验证脚本
4. **Docker 集成**：支持在 `bluebell-ai` 容器中执行测试
5. **测试文档**：编写测试计划和执行指南

## 功能 (Capabilities)

### 新增功能
- `api-test-infrastructure`: API 测试基础设施（公共函数、HTTP 客户端、JWT 处理）
- `auth-api-tests`: 认证接口测试（注册、登录、刷新 Token、用户信息）
- `article-api-tests`: 文章接口测试（CRUD、搜索、精选、统计）
- `comment-api-tests`: 评论接口测试（创建、查询、更新、删除）
- `like-api-tests`: 点赞接口测试（点赞、取消、状态查询）
- `category-api-tests`: 栏目接口测试（CRUD、文章关联）
- `tag-api-tests`: 标签接口测试（CRUD、文章查询）
- `admin-api-tests`: 管理员接口测试（用户管理、系统统计、性能监控）

### 修改功能
- 无现有功能需求变更

## 影响

- **代码目录**: 新增 `tests/api/` 目录及测试文件
- **脚本**: 新增 `scripts/run_tests.sh` 用于 Docker 环境测试执行
- **依赖**: 可能需要添加测试依赖（如 testify）
- **CI/CD**: 可集成到持续集成流程中
- **开发流程**: 开发人员可在提交前本地运行测试验证
