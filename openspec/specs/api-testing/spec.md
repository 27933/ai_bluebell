## 新增需求

### 需求:必须验证服务可用性
测试执行前必须验证服务是否可用。

#### 场景:服务健康检查
- **当** 执行测试前
- **那么** 必须能够访问 `http://localhost:8084/ping` 并返回 200

### 需求:必须测试认证模块
测试必须覆盖用户注册、登录、Token刷新、用户信息获取和更新功能。

#### 场景:认证测试覆盖
- **当** 运行 auth_test.go 时
- **那么** 必须测试注册成功、用户名已存在、密码不匹配、缺少字段
- **且** 必须测试登录成功、密码错误、用户不存在
- **且** 必须测试Token刷新成功、无效Token
- **且** 必须测试获取用户信息（未登录/已登录）
- **且** 必须测试更新用户信息

### 需求:必须测试文章模块
测试必须覆盖文章的创建、查询、更新、删除功能。

#### 场景:文章测试覆盖
- **当** 运行 article_test.go 时
- **那么** 必须测试获取文章列表、精选文章、文章详情
- **且** 必须测试创建文章（未登录/已登录）
- **且** 必须测试更新文章、删除文章

### 需求:必须测试标签模块
测试必须覆盖标签的创建、查询、更新、删除功能。

#### 场景:标签测试覆盖

- **那么** 必须测试获取所有标签
- **且** 必须测试创建标签、更新标签、删除标签

### 需求:必须测试栏目模块
测试必须覆盖栏目的创建、查询、更新、删除功能。

#### 场景:栏目测试覆盖
- **当** 运行 category_test.go 时
- **那么** 必须测试获取栏目列表、栏目详情
- **且** 必须测试创建栏目、更新栏目、删除栏目

### 需求:必须测试评论模块
测试必须覆盖评论的创建、查询、更新、删除功能。

#### 场景:评论测试覆盖

- **且** 必须测试获取评论列表
- **且** 必须测试更新评论、删除评论

### 需求:必须测试点赞模块
测试必须覆盖点赞和取消点赞功能。

#### 场景:点赞测试覆盖

- **那么** 必须测试点赞（未登录/已登录）

### 需求:必须测试管理员模块
测试必须覆盖管理员专属功能。

#### 场景:管理员测试覆盖
- **当** 运行 admin_test.go 时
- **那么** 必须测试管理员权限验证
- **且** 必须测试用户管理功能

### 需求:测试必须输出清晰的结果
测试执行必须输出清晰的通过/失败结果。

#### 场景:测试结果输出
- **当** 测试执行完成后
- **那么** 必须显示通过的测试数量
- **且** 必须显示失败的测试数量（如有）
- **且** 失败的测试必须显示具体原因
