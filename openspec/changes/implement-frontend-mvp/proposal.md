## 为什么

后端 API 已完成并通过全量测试，现在需要实现前端应用以提供完整的用户体验。前端采用 Vue 3 + TypeScript，实现 MVP 阶段的所有核心功能，支持游客浏览、用户登录注册、作者发布文章、评论互动等关键业务流程。

## 变更内容

1. **新增前端项目**：基于 Vue 3 + TypeScript 构建完整前端应用
2. **核心功能实现**：首页、文章详情、登录注册、作者仪表板、文章编辑、搜索、评论等
3. **后端 API 权限修改**：
   - 评论列表接口允许未登录用户访问（访客可查看评论）
   - 文章创建权限校验：仅 author 角色可创建文章
4. **HTML 原型调整**：删除收藏功能、移除粉丝数统计、修正登录为用户名方式
5. **设计文档完善**：编写完整的设计规范、API映射、流程图

## 功能 (Capabilities)

### 新增功能

- `frontend-mvp`: Vue 3 + TypeScript 前端应用，包含游客浏览、用户认证、内容管理、互动功能
- `auth-flow`: 用户注册、登录、Token刷新、个人资料管理
- `article-browsing`: 首页列表、搜索、标签筛选、文章详情展示
- `article-creation`: 作者创建/编辑/删除文章，Markdown编辑器，草稿自动保存
- `comment-system`: 评论展示、新增、删除（支持嵌套回复）
- `like-system`: 文章和评论点赞/取消点赞，实时计数更新
- `dashboard`: 作者仪表板，统计卡片，文章管理表格，阅读趋势图
- `error-handling`: 统一的 Toast 错误提示机制
- `responsive-ui`: 现代化 UI 设计，暂时仅支持桌面版本

### 修改功能

- `comments-api`: **BREAKING** - `GET /comments` 不再要求 Authorization header，允许访客访问
- `article-creation-api`: 增强权限校验，确保 reader 角色无法创建文章

## 影响

**前端**：
- 新增完整前端项目结构（组件、页面、服务、状态管理等）
- 引入依赖：CodeMirror（Markdown编辑）、ECharts（图表）、UI 组件库等
- 部署：跟后端一起部署到同一域名

**后端**：
- 修改 2 个 API 的权限逻辑
- 需要确认文章创建权限检查已正确实现

**用户体验**：
- 游客可完整浏览内容、搜索、查看评论，但无法评论和点赞
- 注册用户可评论和点赞，但无法创建文章（需管理员升级为 author）
- 作者可完整管理自己的内容
