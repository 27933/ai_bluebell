## 新增需求

### 需求:前端应用提供完整用户体验

系统必须通过 Vue 3 + TypeScript 前端应用为所有用户类型（访客、读者、作者、管理员）提供完整的用户交互界面，支持浏览、搜索、认证、发布、评论、点赞等核心功能。

#### 场景:访客浏览文章

- **当** 未登录用户访问应用首页
- **那么** 系统显示文章列表、搜索框、标签筛选，并允许用户点击查看文章详情、评论

#### 场景:用户注册和登录

- **当** 访客填写用户名、邮箱、密码并提交注册表单
- **那么** 系统创建新用户（Reader角色）并自动登录
- **当** 用户点击登录并输入用户名、密码
- **那么** 系统验证凭证并返回 access_token + refresh_token

#### 场景:作者发布文章

- **当** Author 角色用户点击"写文章"进入编辑页面
- **那么** 系统显示 Markdown 编辑器、实时预览、保存草稿、发布按钮
- **当** 用户点击"发布"
- **那么** 系统上传文章，状态为 published，并重定向到文章详情页

#### 场景:用户互动（评论和点赞）

- **当** 已登录用户访问文章详情页面
- **那么** 系统显示评论列表和点赞按钮，用户可以发表评论或点赞
- **当** 用户点击"点赞"按钮
- **那么** 系统立即更新计数，按钮状态变为已赞

### 需求:系统必须支持多种用户角色

系统必须根据用户角色（visitor/reader/author/admin）显示不同的导航菜单项、功能按钮和可操作的页面。

#### 场景:Reader 用户菜单

- **当** Reader 角色用户登录
- **那么** 导航栏显示：我的主页、个人资料、退出登录（不显示"写文章"、"仪表板"）

#### 场景:Author 用户菜单

- **当** Author 角色用户登录
- **那么** 导航栏显示：我的主页、写文章、仪表板、个人资料、退出登录

### 需求:系统必须实现仪表板

Author 用户必须能访问仪表板查看文章统计、管理文章列表、查看阅读趋势。

#### 场景:仪表板统计卡片

- **当** 作者访问仪表板
- **那么** 系统显示统计卡片：已发布文章数、总浏览量、获赞总数
- **当** 用户点击折线图时间段选择器
- **那么** 系统加载相应时间段的阅读趋势数据并重新绘制图表

### 需求:系统必须提供错误处理

所有错误必须通过 Toast 提示展示，用户可以一目了然地了解操作的成功或失败。

#### 场景:网络错误提示

- **当** 网络请求失败
- **那么** 系统显示 Toast："网络连接失败，请重试"

#### 场景:权限错误提示

- **当** Reader 用户尝试创建文章
- **那么** 系统返回 1013 错误，显示 Toast："你没有权限执行此操作"
