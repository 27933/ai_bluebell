## 新增需求

### 需求:管理后台使用独立布局
管理后台必须使用独立的 `AdminLayout` 组件，与普通用户的 `DefaultLayout` 完全隔离，包含侧边栏导航和顶部栏。

#### 场景:访问管理后台页面
- **当** 用户访问 `/admin/*` 任意路径
- **那么** 页面必须渲染 AdminLayout（侧边栏 + 内容区），而不是普通用户的顶部导航布局

#### 场景:侧边栏显示导航菜单
- **当** AdminLayout 渲染时
- **那么** 侧边栏必须包含以下菜单项：系统统计、用户管理、文章管理

### 需求:非 admin 用户禁止访问管理后台
`/admin/*` 路由必须通过 `beforeEnter` 守卫校验用户角色，非 admin 角色禁止进入。

#### 场景:未登录用户访问
- **当** 未登录用户访问 `/admin/*`
- **那么** 必须跳转到登录页 `/login`

#### 场景:非 admin 已登录用户访问
- **当** role 为 `reader` 或 `author` 的用户访问 `/admin/*`
- **那么** 必须跳转到 `/403` 页面

#### 场景:admin 用户访问
- **当** role 为 `admin` 的用户访问 `/admin/*`
- **那么** 必须正常渲染目标页面

### 需求:管理后台默认重定向
访问 `/admin` 根路径必须自动重定向到统计页。

#### 场景:访问 /admin 根路径
- **当** 用户访问 `/admin`
- **那么** 必须自动重定向到 `/admin/stats`
