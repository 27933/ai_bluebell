## 新增需求

### 需求:展示全站文章列表（含草稿）
管理员必须能查看全站所有状态的文章，包括 draft、published、offline，支持分页。

#### 场景:加载文章列表
- **当** 管理员访问文章管理页
- **那么** 必须调用 `GET /api/v1/admin/articles` 并展示标题、作者用户名、状态、创建时间

#### 场景:按状态筛选
- **当** 管理员选择状态筛选（all/draft/published/offline）
- **那么** 必须重新请求并只显示该状态的文章

#### 场景:按关键词搜索
- **当** 管理员在搜索框输入关键词
- **那么** 必须调用带 `keyword` 参数的接口并更新列表

#### 场景:按作者搜索
- **当** 管理员在作者输入框输入用户名
- **那么** 必须调用带 `author_name` 参数的接口并更新列表

### 需求:设置和取消文章精选
管理员必须能将任意文章设为精选或取消精选。

#### 场景:设置精选
- **当** 管理员点击某篇文章的"设为精选"
- **那么** 必须调用 `PATCH /api/v1/admin/articles/:id/featured`（is_featured: true）并更新该行状态

#### 场景:取消精选
- **当** 管理员点击某篇已精选文章的"取消精选"
- **那么** 必须调用 `PATCH /api/v1/admin/articles/:id/featured`（is_featured: false）并更新该行状态
