## 为什么

MVP 上线后，站点缺乏管理手段：无法从界面封禁用户、修改角色、设置精选文章或查看全站数据。后端 admin API 已全部就绪，缺的是对应的前端管理后台。

## 变更内容

- 新增独立的 `/admin` 路由布局（与普通用户界面隔离）
- 新增用户管理页面：查看用户列表、修改角色、封禁/解封、批量操作
- 新增文章管理页面：查看全站文章（含草稿）、设置精选、按状态/关键词/作者筛选
- 新增系统统计页面：全站数据概览 + 每日新增趋势图
- 新增 admin 路由守卫：非 admin 角色访问 `/admin/*` 自动跳转 403

## 功能 (Capabilities)

### 新增功能

- `admin-layout`: 独立的管理后台布局（侧边栏导航、顶部栏、与主站布局完全隔离）
- `admin-user-management`: 用户列表、角色修改、状态封禁/解封、批量操作
- `admin-article-management`: 全站文章管理、精选设置、多条件筛选
- `admin-stats-dashboard`: 系统概览统计卡片 + 每日新增趋势图表

### 修改功能

（无——后端 API 已在独立变更中完成，本次变更纯前端）

## 影响

- **前端新增文件**：`src/views/admin/` 目录下新增 4 个页面组件 + 1 个布局组件
- **前端路由**：`src/router/index.ts` 新增 `/admin` 路由组及守卫
- **前端 API 层**：`src/api/admin.ts` 新增 admin 相关请求函数
- **后端**：无需改动（`GET /admin/articles` 已在本次迭代中完成 JOIN 改造）
