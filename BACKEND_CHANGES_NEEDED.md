## 🔴 关键修改项（MVP 阶段需完成）

### 修改1：评论列表 API 权限修改

**API 端点**: `GET /comments`

**当前状态**: 需要 `Authorization` header（仅登录用户可访问）

**需要修改**: 允许未登录用户（访客）访问

**具体需求**:
- 请求不需要包含 `Authorization` header
- 返回相同的评论列表数据
- 访客无法发表评论（发表时需要 token）
- 已登录用户调用时行为保持不变

**验证方案**:
```bash
# 访客调用（无token）
curl http://localhost:8084/api/v1/comments?article_id=123

# 应该返回 1000 success + 评论列表，而不是 1006 需要登录
```

**影响范围**:
- 前端可以显示文章评论给所有用户（包括未登录访客）

---

### 修改2：文章创建权限验证确认

**API 端点**: `POST /articles`

**当前状态**: 需要确认权限检查是否已实现

**需要确认**: Reader 角色用户调用时返回 1013（无权限错误）

**具体需求**:
- 仅 `author` 和 `admin` 角色可以创建文章
- `reader` 角色调用 `POST /articles` 应返回 HTTP 403 或错误码 1013
- 错误消息应说明权限不足

**验证方案**:
```bash
# Reader 用户调用
curl -X POST http://localhost:8084/api/v1/articles \
  -H "Authorization: Bearer <reader_token>" \
  -H "Content-Type: application/json" \
  -d '{"title": "test", "content": "test"}'

# 应该返回 1013 无权限错误，而不是成功
```

**影响范围**:
- 前端菜单中 Reader 用户看不到"写文章"按钮
- 后端作为第二层防护，确保权限正确
